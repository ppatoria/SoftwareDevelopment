Readme for Mdbg Winforms GUI sample.

 

(This sample is an extension to the Mdbg sample and is under the mdbg eula. See eula.txt.)

 

Instructions for Mdbg Winforms GUI sample.

 

1)      Install and build the Mdbg sample (http://blogs.msdn.com/jmstall/archive/2004/09/30/236281.aspx). “$” refers to the root of the Mdbg installation.

2)      Unzip the files to the $\Extensions\Gui.  This will overwrite the original GUI that shipped in the sample with the new gui.  

3)      Ensure that the new gui builds.

4)      Go to “$\versionInfo.cs” and change then Assembly version attribute from:

[assembly:System.Reflection.AssemblyVersion("1.0.*")]

to

[assembly:System.Reflection.AssemblyVersion("1.0.1000.00")]

This is needed because we shouldn’t be using “*” in the assembly version. Without this fix, the gui will build, but when you run it, it will fail to load and bind to the mdbg dlls.

 

To actually run the gui,

1)      Run Mdbg

2)      Load the gui extension. “load gui”

3)      the gui should load, and the winforms should come up.
